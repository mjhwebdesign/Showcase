Light client
“One of my concerns about blockchain is the slow adoption it has in comparison with its benefits. I am aware that technical constraints such as storage requirements or technical knowledge are factors that contribute to the problem. I hope Light Client is going to close this breach.”
Julian Len
aug 12, 2020 · 3 min read

Light Client is an ongoing project whose main goal is to facilitate access to the blockchain for the users without the need of contributing to the consensus, providing mechanisms to check if all the incoming data is consistent. Without being required to participate in the consensus process, the node can only download a piece of the blockchain reducing both the bandwidth and storage required.

Nowadays it’s very common to use third-party services that provide access to an updated database to connect to the network without the need to download the whole blockchain. This is a problem because it adds an unwanted grade of centralization, and also the need for trust in this third-party. Finally, these services result in a single point of failure to a large group of users.

The interesting part of this project is all the engineering challenges involved in its development. It’s a large project which touches a lot of parts of blockchain technology, such as security, consensus, and synchronization protocols.

Another interesting obstacle with the light client is the incentive problem. Many existing Light Client projects failed due to the lack of incentives. There is no “rational” reason a node would want to spend processing power in serving Light Clients instead of processing its blocks.

Finally, security in this technology plays a key role, a node not taking part in the consensus process needs a solid protocol to verify the integrity and consistency of the incoming data.

In conclusion, this project is a milestone for blockchain adoption. Lowering the high requirements of the technology will let users access the whole blockchain database without the need of storing it locally. Then, we can imagine a Dapp or a wallet running in a cellphone, running as a Light Client working as a full node.
