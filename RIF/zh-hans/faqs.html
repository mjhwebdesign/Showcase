普通问题

  • 请解释 IOV、RSK、RIF品牌结构。

    • IOV Labs 作为一家以目标为导向的组织运营，专注于推广和发展下一代开放式区块链架构，支持全球化普惠金融，在新兴技术与大规模应用之间架起桥梁。它是最初开发RSK和RIF平台的组织。更多信息，请访问https://iovlabs.org

  • RIF的愿景是很么？

    • “使区块链技术更简单，更快和可扩展以促进大规模采用”

    RIFOS旨在打造构建完全去中心化互联网的基础。 所有RIF服务旨在促进全球开发人员社区与区块链技术之间的互动。

    事实证明，比特币可以实现点对点，抗审查的价值转移，现在由RSK支持的RIF旨在添加身份、存储、通信、网关、小额支付和网关层，以允许创建去中心化共享经济以保护个人和用户的权利。

    通过提供实现区块链大规模采用所需的基础设施和服务，RIF OS希望贡献一份力量以转变和改善世界各地数百万被金融排斥的人们的生活。

  • RIF的价值主张是什么？

    • 这是否是一个实用工具？如果是的话，它究竟是什么？代币对于销售单独使用RBTC无法出售的硬币是否非常有用。

    这个问题有两个方面，因为RIF既是一组协议标准，又是一个代币。RSK 基础设施框架开放标准（RIF OS）是一套依赖基于智能合约的区块链开放的去中心化基础设施协议，可实现分布式应用程序（dApp）的更快、更轻松和可扩展的开发。

    初始协议包括目录（命名服务协议）、支付（离线支付协议）、数据（数据存储和流媒体协议）、通信（安全路由、会话和加密通信协议）和网关（ 互操作性协议，包括交叉链传输和 oracling 服务）。这些标准还定义了可以作为API和库实现的接口，这些接口可以为任何开发人员抽象和简化去中心化基础设施（区块链和P2P）的使用，即使他们不了解内部工作或去中心化协议的低级功能。 这套协议旨在解决阻止去中心化区块链网络（即比特币、RSK、以太坊等）实现大规模采用的主要问题。从我们的观点来看，两个主要障碍是可持续扩展（链上缩放是可能的，但导致验证节点的维护成本更高，因此导致集中化）和开发人员可用性（开发人员可能需要几个月才能学习如何使用该技术，而且即使掌握该技术，由于缺乏更高级别的协议和可重用的组件，构建去中心化应用程序的效率非常低）。

    遵循RIF OS的指导原则，我们正在构建一系列基于区块链的P2P平台作为RNS，这是第一个启动的在RSK上实施RIF目录。RIF Lumino是RIF Payments的首个实施，很快也将推出。在2019年期间，第一个完整堆栈版本将可用。现在我们来讨论RIF OS生态系统中RIF代币的效用。

    第一个也是显而易见的用途是访问RIF OS生态系统中提供的所有服务。为了符合 RSK 基础构架框架，提供商必须至少接受 RIF 代币以换取其服务。最重要的是，某些协议使用RIF代币作为所有服务提供商需要提供的抵押物，以便在RIF市场上提供服务。鉴于这些平台的去中心化性质，这是关键，没有嵌入式保险机制将无法确保最终用户的服务质量。此外，在某些协议上，抵押物与服务提供商之间的合约金额之间的比率将用于在注册的提供商之间动态分配新的服务合约。

    我们还设想在不久的将来，RIF代币的其他用途将在RIF市场周围出现。其中两种最相关的用途是使用 RIF 代币作为发行交易对手无风险稳定资产（即：$RIFDollar、$RIFARS 等）的抵押物，可以使用该代币以稳定资产计价服务价格；以及，使用 RIF 代币结算 RIF 支付中心之间没有共同资产或足够流动性的交易。 我们设想RIF OS将是长期的，作为链外基础设施服务的统一市场，每个启用加密经济的主要智能合约都可以使用，所以尽管RIF代币最初是在RSK网络上创建的，但未来它将可以移植到其他平台，例如以太坊或EOS。它将创造规模经济，并加强去中心化生态系统作为一个整体的抗破坏性，从而进一步实现我们的价值互联网愿景。

  • 为什么IOV Labs选择在RSK之上构建RIF？

    • 基于其受比特币网络保护的第一个开源智能合约平台而享有独特地位，我们选择继续在RSK Smart之上进行构建。RSK智能合约网络和RIF操作系统协议的目标是通过整合智能合约、近即时支付和更高的稳定性，以增加价值并扩展比特币生态系统的功能。这还将使项目可以访问从支付到存储的各种分散基础架构服务，同时加快产品上市时间。

    RSK是比特币网络保护的第一个开源智能合约平台：

    • 接受比特币：比特币是第一个，也是今天最安全的区块链。
    • 安全性：RSK区块链通过合并挖掘进行保护，这意味着RSK可以在双重花费预防和结算终结方面实现与比特币相同的安全性。
    • 可扩展性：RSK每秒最多可进行100项交易，同时不会削弱分散化。
    • 灵活性：即时付款允许新的用例，例如可随时用于零售的商店付款和小额信贷。

    IOV Labs在RSK智能合约网络已构建的坚实基础之上实施了RIF操作系统协议。

  • RIF的市场是什么？

    • RIF市场（MP）是一个允许服务提供商广告其报价，并使用户可以轻松使用它们的系统。它是一组智能合约，用于索引RSK区块链上的报价，而且是一个市场库，其允许应用开发人员直接访问，最后为非技术客户提供友好的用户界面。

    RIF市场将是一个独立的服务，它将用作基于RSK和RIF构建的去中心化共享经济体（DSE）中的服务交换的基础。 RIF市场还将成为消费许多RIF服务的地方，例如持久性存储（文件固定服务）、RNS的域交换服务或RIF付款的“守望台”服务。随着更多内容的发布，还会将其加入其中。

    作为开发人员或最终用户，RIF市场将为您提供一个简单的界面，以支付您使用的所有RIF服务并收集您提供给社区的所有RIF服务。

RIF代币

  • RIF 的代币用途是什么？

    • RIF 代币旨在允许任何代币持有者使用与 RIF OS 协议规范兼容的去中心化基础架构服务。此类服务可能包括第三方开发的基础架构服务，以及可能集成到 RIF OS 市场并同意接受 RIF 代币作为访问/消费服务或应用的手段的任何其他应用。此外，在 RIF OS 市场中提供基础架构服务的组织需要存放一定数量的 RIF 代币，这些代币将被锁定，以保护消费者免受违反服务级别协议的影响。

  • 为何 RIF OS 协议创建了代币，而不是使用 RBTC？

    • 尽管需要 RSK Live Mainnet，且将始终需要，但是执行智能合约在 smartBitcoins (RBTC) 中付款，以保持与比特币生态系统中的奖励完全一致性，RIF OS 协议旨在创建和脱离基础设施层，该基础设施层最初构建在 RSK 生态系统之上，但今后将与其他智能合约支持平台（例如以太坊和 EOS）整合。为做到这一点，重要的是，需要有一种对任何网络都是中性的代币，且无论网络的本地加密货币（RBTC、ETH，EOS 等）的具体价格如何，其价格均根据基础架构服务的提供和需求确定。从用户角度而言，这不会造成任何额外的摩擦，在不久的将来，我们预计，DEXs（去中心化交易所）将提供 RIF OS 协议与 RIF 代币整合的网络本地货币之间的即时转换。RIF 代币的可携带性将创造规模经济，并加强去中心化生态系统作为一个整体的抗破坏性，使价值互联网更接近实现。主要原因是，从长远来看，我们认为，RIF OS 是一个链下基础设施服务的统一市场，可供每个支持智能合约的加密经济（即 RSK、以太网、EOS）使用。在这种情况下，拥有便携式/中性代币是必须的

  • RIF代币和RBTC有什么区别？

    • RSK 基础设施框架开放标准 (RIF OS) 是一套开源和去中心化的基础设施协议，可在统一环境中实现分布式应用程序 (dApp) 的更快、更轻松和可扩展的开发。RIF OS 的显著特点包括：支持去中心化的第三方链下支付网络；去中心化应用程序间的无障碍安全通信 API 集；以及易于开发人员使用的开发界面。RIF OS 服务的访问和支付基于 RIF 代币，所以，开发人员可以访问基于 RIF 协议构建的一套服务，例如，目录、支付、数据馈送、存储和通信，包括第三方开发的基础架构服务，以及任何其他可能部署在 RIF 框架上的应用程序，这些应用程序同意接受 RIF 代币作为访问/消费服务或应用程序的手段。RBTC 是 RSK Live Mainnet 的本地代币，与 BTC 的比例为 1 : 1。其用于支付智能合约的执行费用，与 ETH 用作支付以太坊的媒介相同。通过使用比特币与 RSK 协议之间的桥梁，技术用户可以通过 BTC 之间的转换，以去中心化的方式获得 BTC。较少技术知识的用户可以从支持交易机构获得RBTC，以使用RSK和RSK上运行的所有应用程序。

    合同代码可在 RIF 实验室 github 报告中找到： https://github.com/riflabs/RIF-Token

    以及 RSK 浏览器中部署的合同： https://explorer.rsk.co/address/0x2acc95758f8b5f583470ba265eb685a8f45fc9d5

  • RIF 代币生成合约何时部署和分发？

    • RIF 代币智能合约于 2018 年 11 月 9 日世界标准时间晚上 8:12 左右在 RSK 网络上部署。如 RIF 实验室白皮书的“代币销售”部分所述，已创建和分配共计 10 亿个 RIF 代币

  • 私人销售筹集了多少资金？

    • 正如白皮书中所述，私人销售中筹集的资金约为 22000 BTC（略低，因为4.5% 来自 ETH，与 BTC 相比损失了很多价值）。筹集的资金 100% 用于技术开发和促进采纳，分毫均未分配给 RIF 实验室的团队成员，他们领取标准的市场工资。

  • 私人销售 RIF 代币的价格是多少？

    • 从预售开始到结束，所有价格皆平均设定为每 BTC 14000 RIF

  • 代币销售流程如何？

    • 在私人销售期间可供购买的 RIF 代币由 RIF 实验室直接出售给私人出资者。参与私人销售的出资者已由 RIF 实验室批准，并且其必须办理全面的 KYC 手续，其中包括向 RIF 实验室聘请的专门实施 KYC/AML 流程的第三方提交信息。在取得代币及其可用性时，WP 和 ECA 中有几项关于出资者宗旨和目标的参考文献。

  • 可否提供有关代币赎回过程的信息？

    • 在私人销售期间购买的所有 RIF 代币均由 RIF 实验室在 TGE 日期提供给出资者。这些 RIF 代币在 TGE 日期后一年（赎回期）内将继续供私人销售出资者从 RIF 实验室部署的智能合约中赎回。为了取得其 RIF 代币，私人销售出资者需要按照 RIF 实验室向其发送的赎回说明与相关的智能合约进行交互。这是私人销售出资者在赎回期内取得代币的唯一途径。在赎回期届满时，出资者尚未赎回的任何 RIF 代币将通过智能合约自动返还给 RIF 实验室。

  • 实施何种哪些代币奖励政策？

    • 在私人销售期间购买的RIF代币不受任何禁售安排的约束。但是，如果私人销售出资者分别在3个月、6个月和9个月的期间内在其原钱包中继续存储和持有其初始RIF代币分配，RIF实验室将向其提供奖励代币激励。

  • RIF 代币在何处交易？?

  • 现有多少 RIF 代币，流通供应是多少？

    • RIF 代币智能合约在部署时已发行 10 亿 RIF 代币，但最初只有 3.2 亿代币可供项目早期出资人赎回。余下的代币已锁定在 RIF 代币智能合约中，且在 5 年的时间内以固定间隔解锁。

  • 为何 RIF 代币是 ERC20/ERC677 代币？

    • RIF 代币实际是 RRC677 代币。我们称之为 RRC677，以免与 ERC677 标准混淆。为清晰起见，RSK Live Mainnet 上部署的所有 ERC 编程界面标准均重命名为 RRC。与之对应，RRC677 是 RRC20 标准的改进版本，可以完全与广泛使用的 ERC20 标准兼容，因此，以太坊网络上的所有 ERC20 代币均可以无缝移植到 RSK 网络。

  • 如果切换到钱包上的以太坊网络，我的 RIF 代币会怎样？

    • 您的 RIF 代币将是安全的，但是，由于它们存在于 RSK Live Mainnet上，所以只有选择 RSK Mainnet 作为钱包网络，才能看到这些代币。

  • 您可否将 RIF 保持在一个以太坊地址？

    • 不能。RIF 代币仅存在于 RSK Live Mainnet上。以太坊和 RSK 地址在没有激活校验和时看起来是一样的。在激活校验和时，RSK 和以太坊将把不同的字母变为大写。要检查您的地址所属网络，可以访问此网站： https://address-validator.rsk.co

  • 我在钱包里需要 RBTC 来发送 RIF 代币吗？

    • 是的，RSK Live Mainnet中的任何交易都需要在RBTC 中支付网络费用。日后，RSK 协议将支持包络，网络费用也将通过 RIF 代币或其他代币支付，这些代币将在 RBTC 的保护下转换，为最终用户创造更加无缝的体验。

    有关更多信息，请访问： https://developers.rsk.co/tutorials/send-tokens-through-metamask/

  • 如果在其他区块链浏览器中发现 RIF（例如，Etherscan），我可否信任？

    • RIF 代币仅部署在 RSK Live Mainnet 上。您可以在RSK 浏览器中看到它。 如果在不同的区块链发现 RIF 代币，请在“联系我们”版块进行举报。

  • 我可否在 MyCrypto 提供的 RBTC 地址上接收 RIF 代币？

    • 可以，您可以使用 RSK 地址接收 RIF 代币。

    有关如何使用MyCrypto的说明，请访问 https://github.com/rsksmart/rskj/wiki/RSK-with-MyCrypto

    • RIF OS 包含以下服务：目录、支付、存储、通信和网关。

  • RIF 目录是什么？

    • RIF目录是 RIF 生态系统中的身份和信誉层，其中 RNS 是其主要组成部分之一。 它将为用户和节点提供统一的 API 和库，以与所有主要的自主权身份协议进行交互。RIF 目录旨在允许用户轻松控制其 ID 以在去中心化经济体中进行交互，同时建立自主权声誉，将使用户（尤其是不属于传统金融系统的用户）能够参与未来的去中心化数字经济。RIF 目录的主要目标是保护用户的个人数据，使用户能够管理可以访问数据的人员并完全控制自己的声誉，因此用户可以使用它与多个市场和平台进行交互，自由地从一个市场转移到另一个市场，而不会失去其业绩记录、联系人和社会价值。

    目录规范
  • 什么是 RIF 名称服务？

    • RIF 名称服务（RNS）是允许用户拥有任意区块链中可读域的去中心化服务。可用于识别其他个人资源，例如支付、ID、存储或通信地址。 有关更多信息，请访问 rifos.org/nameservice

  • 我是否可以在 RNS 中注册域名然后在二级市场上出售？

    • 在RNS中注册域名的任何人都可以直接或使用第三方二级市场出售域名。IOVLabs目前尚未参与此类市场的开发，但对于全球开发人员而言，为RNS域提供去中心化且易于使用的市场可能是一个好主意。关注RNS的全部细节？

    RIF名称服务(RNS)的目的在于通过一种架构为用户提供更友好的体验，这种架构可以提供人类能够理解的名称或别名来识别区块链的地址。可用于识别其他个人资源，例如支付或通信地址。 集中访问与人类可理解名称相关联的多个资源可改善区块链平台用户体验。随着通过添加名称解析服务或“别名”实现“易用性”，出错概率可大幅降低。由于资源名称可能会随时间而变化，因此系统需要灵活支持频繁更改。到目前为止，虽然RIF名称服务仅支持构建在RSK网络上的地址，但最近用户可以管理多种类型的货币和资产。

    关于 RNS 的更多信息，请访问：https://www.rifos.org/rif-name-service/

  • 什么是 RIF 支付？

    • RIF 支付是一种协议，该协议允许访问任何脱链支付网络，支持 RSK 网络上部署的任何代币，从而实现快速，高度可扩展的低成本脱链支付。通过在不同的支付网络之间创建一个抽象层，并允许跨代币和跨网络交易，其目标是允许用户与Lumino、Lightning和Raiden等多个跨区块链链外网络进行无缝交互。

  • Lumino是什么？

    • RIF Lumino Network 由 RSK 侧链网络提供支持，是比特币区块链的第三层解决方案，可为部署在 RSK 网络上的任何代币提供对比特币友好的链外支付功能，从而将成本降低了近一个数量级， 并提供即时付款处理速度。

    Lumino 网络的优势：

    • 低成本
    • 更快的确认时间
    • 支持任意 RSK 部署的代币
    • 跨代币支付
    • 友好的 Lightning 网络
      • 发票系统与 LN 的兼容
      • 跨网络支付
  • Lumino 基本信息

    • https://www.rifos.org/rif-lumino-network/.

  • 关于向公众推出Lumino，您有什么计划？

    • RIF Lumino 网络已经向公众开放。（有关更多信息，请访问： https://www.rifos.org/rif-lumino-network/) 说到这一点，制作 Lumino 和方便用户使用的价值互联网是 RIF 的重点工作之一。因此，Lumino已经与RIF命名服务（RNS）集成，大大简化了非技术用户的可用性。 我们目前还在开发 Lumino 轻客户端和开发库，以方便与钱包和交易所的集成。 IOV Labs团队还致力于为愿意将RIF Lumino用于其业务需求的银行和组织开发解决方案。

  • Lumino 网络的各种 KPI 有哪些：每秒区块量、到达终点的时间、每秒交易量和每笔交易成本？人们是否已可以在Lumino基础上构建服务？哪些项目正在该基础上构建？

    • Lumino 每秒可以实现的交易数量主要取决于实际的网络拓扑和参与者在其渠道中锁定的钱币数量。此外，从技术角度看，网络中参与的计算机的带宽和延迟也是提供响应系统的关键。此外，网络的容量将取决于其用户的网络使用模式。似乎还有太多未知因素。但是，我们可以模拟从小型网络到越来越大的网络的某些预期模式，并获得关于网络增长和成功支付数量、支付结算时间和平均成本的有用指标。考虑到 RSK Labs 已经为 RSK 开发的可扩展性改进提议的合并，我们从获得的指标中看到，Lumino 可以扩展到 6,000 万活跃用户，在此情况下成本和响应时间相对于其他支付网络仍将具有竞争力。为了扩大规模，我们看到了需要解决的资源瓶颈。我们期望学术界正在开发的其他第三层解决方案，例如委托链，能够与Lumino互操作并对其进行补充。 有几个项目将钱包和解决方案与 Lumino 整合在一起，该整合在接下来几个月内准备就绪后即会公布。

  • 我怎样才能加入 Lumino 网络？

    • 如果您希望成为网络的一部分，Lumino存储库是开放的，您可以在存储库中找到节点配置和管理的相关说明。

  • 现在Lumino已开始运营，下一步行动是什么？

    • 我们正在研究即将推出的新RIF支付组件以及RIF存储协议。到今年年底，我们计划使用全套RIF OS服务，这将展示完整堆栈如何协同工作。

  • Lumino如何与Lightning网络竞争？

    • 我们不认为Lightning是竞争对手，而是补充。通过双Lightning / Lumino节点，人们将能够为smartBitcoin进行比特币的原子交换，大大简化了RSK网络的使用。 另一方面，Lightning目前只是比特币，而RIF Lumino Payments将为在RSK网络上的任何代币实现链外支付。想象一下，即时支付稳定资产的可能性，这些资产与法定货币紧密相连，与比特币完全整合，成本不到一美分。这可以为全世界的金融科技创造一个完美的环境，将金融体系的竞争推向一个全新的高度。

  • 你们是否正在考虑在 RIF 中实施与去中心化身份相关的解决方案？

    • 是的。与 RIF 钱包库一起，RIF Identity 是 RIF 库最重要的组件之一。它提供了RSK网络上锚定身份的基础，以及此后签署和交易所事件的证明，这些证明之后可用于构建声誉模型。我们正在与该领域的顶级专家（Sovrin、uPort和其他人）进行讨论，以制定一项共同标准。 我们还与微软建立了合作关系，这是 ID2020 工作的一部分。此外，我们正在与非政府组织比特币阿根廷、美洲开发银行和埃森哲（另一个 ID2020 成员）合作，以围绕布宜诺斯艾利斯贫民窟的信誉身份创建和实施第一个包容性金融生态系统。

  • 什么是RIF存储？

    • RIF存储是一种不受审查的，无权限且去中心化的存储协议。它提供了一个统一的接口，可以轻松地在多个去中心化存储网络（例如IPFS或Swarm）之间进行连接和切换。我们正在向这些系统添加激励措施，以确保抵抗微不足道的攻击，提高可用性，并使参与者因参与网络而获得奖励。

    作为最终用户，RIF存储使您可以收回数据的所有权。即使您正在使用的服务停止存在，您也可以检索所有内容。

    作为开发人员/服务创建者，RIF存储使您不必担心应用程序的存储，它还提供了现成的内容交付服务，因此您可以扩展应用程序而无需投资基础架构。

    借助内置的激励措施，任何人都可以成为存储提供商，并将未使用的存储提供给其他人并从中获得收益。

    IOV Labs 正在努力建立一个统一的 API 以存储和检索文件，并支持多个存储网络。这是 RSK 数据存储协议。对于第一个网络提供商，我们查看了现有的解决方案（Swarm、IPFS、Storj、Sia），并决定在 Swarm 的基础上构建。其中大多数协议实施以下一种变体：上传的文件分成块并在网络中分布。请求文件时，将检索并组合所有块。参与该网络的每个节点都跟踪为支付目的而存储/提供的数据。我们目前正在进行的创新是简化激励模型和证明机制。当然，RSK 数据存储将与其他 RIF 服务（例如用于检索命名文件并允许可变性的 RNS）或 RSK 付款进行集成以提供激励。而且将来我们将在相同的 RIF 存储 API 和用户界面下促进所有成功存储网络的集成，从而用户仅需要从列表中选择提供商即可切换存储网络后端，甚至可以同时在多个网络上存储单个文件。

  • 什么是 RIF 网关？

    • RIF 网关提供了一组协同工作能力协议，旨在实现与外部世界的安全和防篡改交互。它提出了一个接口层，该接口层统一了对oracle服务和跨链集成的访问，为区块链提供了与实现无关的协议，可用于内部和外部数据使用。

    RIF网关利用基于区块链的数据服务的最新研究成果，并实施最广泛认可的行业解决方案，同时提供面向用户的统一API，从而大大降低了实施这些技术的复杂性。它旨在在区块链之间建立桥梁，并允许数据提供商和消费者轻松地进行安全和标准化的数据传输，从而支持各种数据消费、订阅和支付模型。

  • 什么是 RIF 通信？

    • RIF 通信允许对等网络中的各方发现彼此并建立安全的通信通道，并确保匿名性、机密性、完整性和真实性。

    机密性：第三方无法读取发送的消息。
    完整性：防止第三方修改发送的消息。
    真实性：防止冒充端点。
    匿名性：如果启用，网络中则没有人可以识别任何特定消息的发件人或收件人。

    通过使用公共密钥作为发现机制，用户可以在 RIF 目录中发布假名，并且无需记住长公共密钥就可以被发现。RIF 安全通信基础结构协议旨在满足在最终用户、RIFOS 各方或服务之间建立安全链接的需求。
